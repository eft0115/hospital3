# 스마트허브병원 웹사이트 Project Summary

## 문서 목적

이 문서는 스마트허브병원 웹사이트의 **브랜드, 포지셔닝, 웹사이트 컨셉, 대상 사용자, 경험 기준**을 정의한다.  
실제 화면 구조, 컴포넌트, 디자인 토큰, 개발 구조는 후속 문서에서 다루며, 본 문서는 모든 기획 판단의 기준점으로 사용한다.

---

# 1. Project Summary

## 1-1. 배경 및 목적

### (1) 왜 만드는가

스마트허브병원은 단순한 지역 병원 홈페이지로 설명되기에는 실제 콘텐츠와 서비스 방향이 더 넓다.  
134개 블로그 콘텐츠 분석 기준, 병원의 주요 콘텐츠 축은 **배치전·배치후검진, 특수건강검진, 출장·산업체검진, 건강검진, 지역 외래 진료**로 구성되어 있다.

특히 반복적으로 등장하는 핵심 주제는 다음과 같다.

- 배치전 건강검진
- 배치후 건강검진
- 특수건강검진
- 야간근로자 검진
- 출장검진
- 산업체/사업장 검진
- 건설현장, 물류센터, 공단, 학교 등 현장 단위 검진
- 정왕동·시흥·배곧 등 지역 기반 외래 및 검진 수요

즉, 스마트허브병원은 일반적인 “진료과 소개형 병원”이라기보다 **기업, 근로자, 지역 주민의 검진 문제를 해결하는 산업·현장 건강검진 인프라**에 가깝다.

현재 블로그는 검색 유입 자산으로서 기능하고 있지만, 블로그만으로는 다음 정보가 한눈에 정리되기 어렵다.

- 기업 담당자가 출장검진을 어떻게 문의해야 하는지
- 개인 근로자가 배치전검진을 어떻게 예약해야 하는지
- 특수건강검진 대상 여부를 어떻게 확인해야 하는지
- 검진 전 준비물, 금식, 결과 확인, 유효기간은 무엇인지
- 현장별, 산업군별 검진 대응이 가능한지
- 병원이 어떤 검진 운영 역량을 가지고 있는지

따라서 웹사이트는 단순 홍보물이 아니라, **블로그 검색 유입을 실제 문의와 예약으로 전환하는 공식 검진 안내 허브**로 설계되어야 한다.

### (2) 기대 효과

웹사이트 구축을 통해 기대하는 효과는 다음과 같다.

#### 사용자 측면

- 복잡한 검진 정보를 빠르게 이해할 수 있다.
- 기업 담당자와 개인 근로자가 각자의 목적에 맞는 경로로 이동할 수 있다.
- 배치전검진, 특수건강검진, 출장검진 등 생소한 검진 절차를 쉽게 파악할 수 있다.
- 검진 전 준비물, 결과 확인, 문의 방법을 명확히 알 수 있다.
- 병원 방문 전 불확실성과 불안감을 줄일 수 있다.

#### 병원 측면

- 블로그 중심의 분산된 콘텐츠를 웹사이트 안에서 구조화할 수 있다.
- 배치전검진, 특수건강검진, 출장검진이라는 핵심 강점을 명확히 드러낼 수 있다.
- 기업/기관 문의와 개인 예약을 분리해 전환 효율을 높일 수 있다.
- 지역 외래 병원과 산업검진 인프라라는 이중 정체성을 균형 있게 보여줄 수 있다.
- 네이버 블로그 → 공식 웹사이트 → 문의/예약으로 이어지는 전환 흐름을 만들 수 있다.

## 1-2. 브랜드 철학

### (1) 네이밍의 의미

> 본 문서에서의 네이밍 해석은 웹사이트 기획을 위한 브랜드적 해석이다.

**스마트허브병원**이라는 이름은 두 가지 의미를 가진다.

#### Smart

단순히 기술적이라는 의미보다, 사용자가 복잡하게 느끼는 의료·검진 과정을 **더 빠르고, 더 명확하고, 더 효율적으로 안내하는 방식**을 의미한다.

스마트함은 화려한 디지털 기능보다 다음과 같은 경험으로 드러나야 한다.

- 필요한 정보를 빠르게 찾을 수 있음
- 기업과 개인의 동선을 명확히 구분함
- 검진 절차를 한눈에 이해할 수 있음
- 준비물, 금식, 결과, 비용, 대상 여부 등 실질적 질문에 답함
- 문의까지의 이동이 짧고 명확함

#### Hub

허브는 병원이 단순히 진료가 이루어지는 한 장소가 아니라, **기업, 근로자, 지역, 현장, 검진, 진료를 연결하는 중심점**이라는 의미다.

스마트허브병원은 다음을 연결한다.

- 기업 담당자와 검진 운영
- 근로자와 배치전·특수건강검진
- 산업 현장과 의료 서비스
- 지역 주민과 외래 진료
- 검색 정보와 실제 예약/문의
- 병원 방문 검진과 출장검진

따라서 웹사이트는 “병원 소개 페이지”가 아니라 **사용자가 자신의 검진 문제를 어디서부터 해결해야 하는지 안내하는 연결 허브**가 되어야 한다.

### (2) 3가지 핵심가치

## 2-1. 편의성

### 정의

편의성은 단순히 예약 버튼이 잘 보이는 것을 의미하지 않는다.  
스마트허브병원에서의 편의성은 **사용자가 복잡한 검진 절차를 스스로 이해하고, 자신의 상황에 맞는 다음 행동으로 쉽게 이동할 수 있는 것**이다.

### 사용자 경험에서의 표현

- 기업 담당자와 개인 근로자의 경로를 첫 화면에서 분리한다.
- 배치전검진, 특수건강검진, 출장검진의 절차를 단계별로 보여준다.
- 금식, 준비물, 결과 확인, 유효기간, 가능 시간 등 자주 묻는 질문을 쉽게 찾게 한다.
- 모바일에서도 전화, 카카오톡, 예약, 문의로 빠르게 이동할 수 있게 한다.
- 사용자가 “내가 지금 무엇을 해야 하는지”를 1~2분 안에 파악할 수 있게 한다.

### 웹사이트 적용 방향

- 메인 CTA 이원화: `기업 출장·특수검진 문의` / `개인 배치전검진 예약`
- 검진별 체크리스트 제공
- FAQ 우선 배치
- 검진 절차 타임라인 제공
- 모바일 하단 고정 CTA 적용
- 블로그 콘텐츠를 단순 게시판이 아니라 가이드형 콘텐츠로 재구성

## 2-2. 문제해결력

### 정의

문제해결력은 의료 서비스를 나열하는 것이 아니라, 사용자가 실제로 겪는 검진 관련 문제를 해결하는 능력이다.

스마트허브병원이 해결해야 하는 문제는 다음과 같다.

- 사업장 검진을 어디에 맡겨야 할지 모르는 기업 담당자의 문제
- 배치전검진이 필요한데 절차를 모르는 근로자의 문제
- 특수건강검진 대상 여부를 이해하기 어려운 문제
- 출장검진 가능 여부, 인원, 일정, 결과 전달 방식이 불명확한 문제
- 병원 방문 전 준비물과 소요시간을 몰라 생기는 불안
- 법정검진과 일반검진의 차이를 이해하기 어려운 문제

### 사용자 경험에서의 표현

- “무엇을 제공하는가”보다 “어떤 문제를 해결하는가”를 먼저 보여준다.
- 기업 담당자용 정보와 개인 근로자용 정보를 분리한다.
- 산업군별, 상황별 안내를 제공한다.
- 검진 과정에서 발생하는 행정적 질문을 미리 답변한다.
- 블로그에 흩어진 정보를 웹사이트 안에서 체계적인 가이드로 묶는다.

### 웹사이트 적용 방향

- `출장검진`, `배치전·배치후검진`, `특수건강검진`을 독립된 핵심 메뉴로 구성
- 각 페이지에 “이런 분께 필요합니다” 섹션 삽입
- 기업 담당자용 프로세스와 개인 근로자용 프로세스 분리
- 검진 전/당일/이후 흐름을 명확히 안내
- 산업군별 랜딩 섹션 제공: 건설현장, 물류센터, 제조업, 공단, 학교/기관 등

## 2-3. 안정감

### 정의

안정감은 병원 웹사이트에서 가장 중요한 감각 중 하나다.  
특히 건강검진, 특수건강검진, 배치전검진은 사용자가 절차를 정확히 이해해야 하고, 기업 담당자는 신뢰할 수 있는 운영 파트너를 찾아야 한다.

따라서 안정감은 다음 요소를 통해 만들어져야 한다.

- 의료적 신뢰
- 행정적 명확성
- 절차의 예측 가능성
- 과장 없는 설명
- 빠른 문의 연결
- 깔끔하고 정돈된 정보 구조

### 사용자 경험에서의 표현

- 사용자가 검진 과정에서 헷갈리지 않도록 단계별 정보를 제공한다.
- 불필요한 광고성 표현보다 사실 중심의 안내를 사용한다.
- 병원의 장비와 진료과는 신뢰 보강 요소로 배치하되, 메인 메시지를 흐리지 않는다.
- 검진 전후에 무엇이 필요한지 명확히 알려준다.
- 기업 담당자가 “여기에 맡기면 일정과 운영이 안정적으로 진행되겠다”고 느끼게 한다.

### 웹사이트 적용 방향

- 화이트/블루/그레이 기반의 명확한 의료 UI
- 과한 애니메이션보다 안정적인 카드형 레이아웃
- 프로세스, FAQ, 문의 경로를 반복적으로 제공
- 장비·진료과 정보는 신뢰 보강 섹션으로 사용
- 공포 마케팅, 과장된 최고 표현, 지나친 감성 카피 배제

### (3) 브랜드 태도

스마트허브병원의 브랜드 태도는 다음과 같이 정의한다.

> 빠르게 안내하되 가볍지 않게,  
> 전문적으로 설명하되 어렵지 않게,  
> 현장의 문제를 실질적으로 해결하는 태도.

브랜드는 사용자를 설득하기보다 **안내**해야 한다.  
사용자가 이미 검색을 통해 불편함과 궁금증을 가지고 들어오기 때문에, 웹사이트는 화려한 홍보보다 명확한 해답을 제공해야 한다.

### 브랜드가 취해야 할 태도

- 친절하지만 과도하게 감성적이지 않다.
- 전문적이지만 어려운 의학용어에 갇히지 않는다.
- 빠른 전환을 유도하지만 압박하지 않는다.
- 병원의 장점을 말하되 과장하지 않는다.
- 기업 담당자에게는 운영 안정성을, 개인 근로자에게는 절차의 쉬움을 제공한다.

### 피해야 할 태도

- “최고”, “최대”, “무조건” 같은 과장 표현
- 공포감을 조성하는 질환 마케팅
- 진료과와 장비만 나열하는 방식
- 기업과 개인 사용자를 한 동선에 섞는 방식
- 블로그 글을 단순히 많이 쌓아두는 게시판식 구성

## 1-3. 브랜드 방향성

### 포지셔닝

스마트허브병원의 웹사이트 포지셔닝은 다음과 같이 설정한다.

> 정왕동 지역 병원을 넘어,  
> 기업과 근로자를 연결하는 산업·현장 건강검진 허브.

또는 더 간결하게:

> 사업장과 현장으로 찾아가는 건강검진 인프라.

### 포지셔닝 해석

스마트허브병원은 지역 외래 진료 기능을 가지고 있지만, 블로그 콘텐츠와 검색 자산을 기준으로 보면 핵심 경쟁력은 산업검진 영역에 있다.

따라서 웹사이트의 상위 포지셔닝은 다음과 같이 잡는다.

- 지역 기반 의료기관
- 산업체·사업장 검진 파트너
- 배치전·특수건강검진 안내 허브
- 출장검진 운영이 가능한 현장 대응형 병원
- 기업 담당자와 개인 근로자를 모두 연결하는 검진 플랫폼

### 커뮤니케이션 톤

스마트허브병원의 커뮤니케이션 톤은 **명확한 안내형 의료 톤**이어야 한다.

#### 기본 톤

- 명확하게
- 쉽게
- 빠르게
- 책임감 있게
- 과장 없이
- 실무적으로

#### 기업 담당자 대상 톤

```txt
검진 일정, 대상, 인원, 항목, 결과 전달까지
사업장 검진 운영에 필요한 과정을 명확하게 안내합니다.
```

기업 담당자에게는 감성보다 **운영 신뢰**가 중요하다.

#### 개인 근로자 대상 톤

```txt
배치전검진이 처음이어도 필요한 준비와 절차를 쉽게 확인할 수 있습니다.
```

개인 근로자에게는 전문성보다 **불안 해소와 절차 안내**가 중요하다.

#### 지역 주민 대상 톤

```txt
검진과 외래 진료를 가까운 곳에서 편하게 연결합니다.
```

지역 주민에게는 **접근성, 편안함, 신뢰감**이 중요하다.

## 1-4. 웹사이트 컨셉

### 핵심 컨셉 선언

> 스마트허브병원 웹사이트는  
> 기업과 근로자가 복잡한 검진 절차를 빠르게 이해하고,  
> 병원 방문 또는 출장검진 문의까지 명확하게 이동할 수 있도록 돕는  
> 산업·현장 건강검진 허브이다.

더 짧은 컨셉 문장:

> 찾아가는 건강검진 인프라.

또는:

> 현장과 병원을 연결하는 검진 안내 허브.

---

### 컨셉을 구성하는 3축

## 1. 시간축

스마트허브병원의 웹사이트에서 시간은 매우 중요한 요소다.  
사용자는 대부분 “언제까지 검진을 받아야 하는지”, “오늘 가능한지”, “결과가 언제 나오는지”, “검진 전에 무엇을 준비해야 하는지”를 알고 싶어 한다.

따라서 웹사이트는 검진을 시간 흐름에 따라 안내해야 한다.

### 시간축의 핵심 흐름

```txt
검진 필요 발생
→ 대상 여부 확인
→ 준비물/금식 확인
→ 예약 또는 문의
→ 검진 진행
→ 결과 확인
→ 필요 시 사후관리
```

### 웹사이트 적용

- 배치전검진 페이지에 당일검진/결과 확인/유효기간 안내
- 특수건강검진 페이지에 정기성, 대상자, 검진 주기 안내
- 출장검진 페이지에 문의부터 결과 전달까지의 일정 흐름 안내
- FAQ에서 시간 관련 질문 우선 노출
- 메인에서 “지금 해야 할 일”을 빠르게 안내

### 사용자가 느껴야 할 시간 감각

```txt
복잡한 검진 절차가 빨리 정리된다.
기한이 있는 검진도 어디서부터 시작해야 할지 알 수 있다.
문의하면 일정 조율이 빠르게 진행될 것 같다.
```

---

## 2. 공간축

스마트허브병원은 병원이라는 고정된 공간만으로 설명되지 않는다.  
블로그 콘텐츠 분석 기준, 병원의 서비스는 병원 내부를 넘어 **사업장, 건설현장, 물류센터, 공단, 학교, 기관**으로 확장된다.

즉, 이 병원의 공간성은 다음과 같다.

```txt
병원 내부
+ 지역 생활권
+ 산업 현장
+ 기업/기관
+ 온라인 검색 공간
```

### 공간축의 핵심 해석

- 병원은 진료와 검진이 이루어지는 중심 공간이다.
- 현장은 검진 수요가 발생하는 실제 공간이다.
- 웹사이트는 병원과 현장을 연결하는 온라인 허브다.
- 블로그는 검색 유입 공간이고, 웹사이트는 전환 공간이다.

### 웹사이트 적용

- 출장검진 가능 공간을 시각적으로 보여준다.
- 산업군별 카드 UI를 구성한다.
- 지역 키워드는 단순 나열이 아니라 서비스 가능 범위로 해석한다.
- 병원 방문 검진과 출장검진을 구분해 안내한다.
- 오프라인 이동 전 온라인에서 준비가 끝나도록 구성한다.

### 사용자가 느껴야 할 공간 감각

```txt
이 병원은 병원 안에서만 기다리는 곳이 아니라,
현장의 검진 문제까지 대응할 수 있는 곳이다.
```

---

## 3. 철학축

스마트허브병원의 철학축은 **의료 서비스 나열이 아니라 검진 문제 해결**이다.

이 웹사이트는 “우리는 이런 진료과가 있습니다”에서 시작하지 않는다.  
대신 “당신이 지금 겪는 검진 문제는 무엇이며, 우리는 그것을 어떻게 해결할 수 있는가”에서 시작해야 한다.

### 철학축의 핵심 원칙

- 사용자의 상황을 먼저 분류한다.
- 기업과 개인의 목적을 분리한다.
- 의료 정보보다 행동 가능한 안내를 우선한다.
- 장비보다 절차를, 진료과보다 문제 해결 흐름을 먼저 보여준다.
- 병원은 사용자의 검진 과정을 정리해주는 파트너로 작동한다.

### 웹사이트 적용

- 첫 화면에서 기업/개인 동선 분리
- 서비스 나열보다 상황별 진입 구조 제공
- 블로그 콘텐츠를 검진 가이드로 재구성
- 검진 전후 행동을 명확히 안내
- 문의 전 사용자의 불확실성을 줄이는 FAQ 제공

### 사용자가 느껴야 할 철학적 감각

```txt
이 병원은 단순히 검사를 해주는 곳이 아니라,
복잡한 검진 과정을 정리하고 해결해주는 곳이다.
```

---

### 사이트가 하지 않는 것

스마트허브병원 웹사이트는 다음을 하지 않는다.

- 일반 병원처럼 진료과만 병렬 나열하지 않는다.
- 기업 담당자와 개인 근로자의 동선을 섞지 않는다.
- 장비 자랑을 메인 메시지로 삼지 않는다.
- 블로그 글을 단순 게시판처럼 쌓아두지 않는다.
- “최고”, “최대”, “무조건” 같은 과장 표현을 사용하지 않는다.
- 질환에 대한 공포를 조장해 예약을 유도하지 않는다.
- 지역 키워드를 무리하게 반복하지 않는다.
- 모든 서비스를 같은 비중으로 보여주지 않는다.
- 사용자가 다음 행동을 찾기 위해 여러 페이지를 헤매게 하지 않는다.
- 감성적 표현만으로 의료적 신뢰를 대체하지 않는다.

## 1-5. 대상 사용자

### 주요 사용자

## 1. 기업/산업체 담당자

기업 담당자는 스마트허브병원 웹사이트의 가장 중요한 B2B 사용자다.

### 주요 니즈

- 출장검진 가능 여부
- 특수건강검진 대상 및 항목 확인
- 검진 가능 인원과 일정 조율
- 사업장별 검진 운영 방식
- 결과 전달 방식
- 협약 및 단체 검진 문의
- 담당자와 빠른 연결

### 웹사이트에서 필요한 경험

```txt
이 병원은 사업장 검진을 운영해본 경험이 있고,
문의하면 일정과 항목을 명확히 조율할 수 있겠다.
```

---

## 2. 개인 근로자

개인 근로자는 배치전검진, 배치후검진, 특수건강검진, 채용검진 등을 검색해 유입될 가능성이 높다.

### 주요 니즈

- 배치전검진을 어디서 받을 수 있는지
- 당일검진이 가능한지
- 준비물은 무엇인지
- 금식이 필요한지
- 결과는 언제 나오는지
- 검진 비용은 얼마인지
- 예약이 필요한지

### 웹사이트에서 필요한 경험

```txt
처음 받아보는 검진이어도 무엇을 준비해야 하는지 바로 알 수 있다.
```

---

## 3. 지역 주민

지역 주민은 내과, 정형외과, 신경외과, 재활치료, 예방접종, 건강검진 등 생활 의료 수요로 유입된다.

### 주요 니즈

- 가까운 병원인지
- 어떤 진료가 가능한지
- 검진과 외래 진료가 연결되는지
- 병원이 신뢰할 만한지
- 예약/문의가 쉬운지

### 웹사이트에서 필요한 경험

```txt
검진뿐 아니라 일상적인 진료도 가까운 곳에서 안정적으로 받을 수 있겠다.
```

---

### 부수적 접점

스마트허브병원 웹사이트는 주요 사용자 외에도 다양한 접점과 연결된다.

- 학교 및 기관 담당자
- 건설현장 관리자
- 물류센터/제조업/공단 담당자
- 협력업체 담당자
- 근로자의 가족 및 보호자
- 네이버 블로그 검색 유입자
- 카카오톡/전화 문의 사용자
- 지역 커뮤니티를 통해 유입되는 사용자
- 기존 환자 또는 검진 경험자의 재방문 사용자

이 접점들은 모두 동일한 목적을 갖지 않는다.  
따라서 웹사이트는 하나의 예약 버튼으로 모든 사용자를 처리하기보다, **상황별 진입 경로**를 제공해야 한다.

---

### 사용자가 기대하는 감각

사용자가 스마트허브병원 웹사이트에서 기대하는 감각은 다음과 같다.

## 명확함

```txt
내가 어떤 검진을 받아야 하는지 바로 알 수 있다.
```

## 빠름

```txt
필요한 정보와 문의 방법을 오래 찾지 않아도 된다.
```

## 신뢰감

```txt
검진 절차와 운영을 잘 아는 병원처럼 느껴진다.
```

## 안정감

```txt
검진 전후 과정이 예측 가능해서 불안하지 않다.
```

## 실무성

```txt
기업 담당자가 실제로 문의하고 일정을 조율하기에 편하다.
```

## 접근성

```txt
개인도, 기업도, 지역 주민도 각자에게 맞는 경로를 찾을 수 있다.
```

## 1-6. 성공 기준

본 웹사이트의 성공 기준은 단순 방문자 수가 아니라 **사용자가 검진 문제를 얼마나 쉽게 이해하고 다음 행동으로 이동했는가**를 중심으로 판단한다.

### 경험 지표 중심 성공 기준

## 1. 경로 명확성

사용자가 첫 화면에서 자신의 목적에 맞는 경로를 빠르게 선택할 수 있어야 한다.

### 확인 지표

- 기업 문의 CTA 클릭률
- 개인 배치전검진 예약 CTA 클릭률
- 메인 이탈률 감소
- 핵심 서비스 카드 클릭률
- 모바일 하단 CTA 사용률

---

## 2. 정보 이해도

사용자가 검진 절차를 이해하고 문의 전 불확실성을 줄여야 한다.

### 확인 지표

- FAQ 페이지 체류시간
- 준비물/금식/결과 안내 섹션 조회율
- 배치전검진 가이드 페이지 재방문율
- 문의 전 사용자가 반복 질문하는 항목 감소
- 전화 문의 시 기본 절차 질문 감소

---

## 3. 전환 효율

블로그에서 유입된 사용자가 웹사이트를 통해 문의 또는 예약으로 이어져야 한다.

### 확인 지표

- 블로그 → 웹사이트 유입률
- 웹사이트 → 전화/카카오/폼 전환율
- 기업 출장검진 문의 수
- 특수건강검진 상담 문의 수
- 개인 배치전검진 예약 수

---

## 4. 브랜드 인식

사용자가 스마트허브병원을 단순 지역 병원이 아니라 **산업·현장 검진을 잘 안내하는 병원**으로 인식해야 한다.

### 확인 지표

- 문의 내용 중 출장검진/배치전검진/특수검진 비중 증가
- 기업/기관 문의 증가
- “검진 절차가 잘 정리되어 있다”는 피드백
- 검색 유입 키워드와 웹사이트 핵심 메뉴의 일치도
- 재문의 및 소개 문의 증가

---

## 5. 안정감

의료 웹사이트로서 신뢰와 안정감을 느끼게 해야 한다.

### 확인 지표

- 과도한 광고성 문구 없이 문의 전환 유지
- 핵심 페이지 이탈률 감소
- 모바일 사용자의 문의 완료율 증가
- 장비/진료과/검진 프로세스 페이지 간 이동률
- 부정확하거나 애매한 문의 감소

# 2. 최종 요약

스마트허브병원 웹사이트는 일반적인 병원 소개형 홈페이지로 기획하면 강점이 흐려진다.  
134개 블로그 콘텐츠가 보여주는 실제 방향은 **배치전검진, 특수건강검진, 출장검진, 산업체검진을 중심으로 한 현장 기반 의료 인프라**다.

따라서 웹사이트의 핵심 방향은 다음과 같다.

```txt
정왕동 지역 병원 홈페이지
→ 기업과 근로자를 위한 산업·현장 건강검진 허브
```

최종 컨셉은 다음과 같이 정의한다.

```txt
스마트허브병원 웹사이트는
기업과 근로자의 복잡한 검진 과정을 명확하게 안내하고,
병원 방문 검진과 출장검진을 연결하는
산업·현장 건강검진 인프라이다.
```
